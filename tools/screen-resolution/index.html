Responsive Breakpoint Guide

Common Breakpoints

Use these standard breakpoints when building responsive layouts. Test your designs at each size to ensure they adapt gracefully.

Breakpoint Range Typical Devices
xs < 576px Mobile phones (portrait)
sm 576px - 767px Large phones, small tablets
md 768px - 991px Tablets (portrait), small laptops
lg 992px - 1199px Laptops, desktops
xl 1200px - 1399px Large desktops
xxl ≥ 1400px Extra large screens

Best Practices

Design mobile-first: Start with the smallest viewport and progressively enhance for larger screens. This ensures your core content is accessible everywhere.

Use relative units: Prefer rem, em, and % over fixed pixels. This allows layouts to scale gracefully.

Test real content: Don't just test with placeholder text. Real content often reveals edge cases and overflow issues.

Check touch targets: On mobile, ensure interactive elements are at least 44×44px for comfortable tapping.

Consider orientation: Test both portrait and landscape modes. Some designs that work in one orientation break in the other.

Device Pixel Ratio (DPR)

Modern devices have high-density displays with DPR values of 2×, 3×, or higher. CSS pixels don't map 1:1 to physical pixels. Use srcset for images and -webkit-device-pixel-ratio media queries when necessary.
