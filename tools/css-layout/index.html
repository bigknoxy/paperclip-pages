CSS Layout Guide

Flexbox vs Grid: When to Use Which?

Flexbox is best for one-dimensional layouts - either a row or a column. Use it for:

  • Navigation bars and menus
  • Centering elements
  • Distributing space between items
  • Card components with variable content
  • Form layouts

CSS Grid is best for two-dimensional layouts - rows and columns together. Use it for:

  • Page layouts (header, sidebar, main, footer)
  • Image galleries
  • Dashboard layouts
  • Complex card grids
  • Magazine-style layouts

Common Flexbox Properties

justify-content

Aligns items along the main axis (horizontal for row, vertical for column). Controls spacing between items.

align-items

Aligns items along the cross axis (vertical for row, horizontal for column). Controls item alignment.

flex-wrap

Allows items to wrap onto multiple lines. Use wrap for responsive layouts.

gap

Sets spacing between flex items. Replaces margin hacks with a clean, modern approach.

Common Grid Properties

grid-template-columns

Defines column tracks. Use repeat() for patterns, fr units for fractions.

grid-template-rows

Defines row tracks. Similar syntax to columns but for vertical tracks.

grid-gap / gap

Sets spacing between grid cells. Modern shorthand for row-gap and column-gap.

grid-auto-flow

Controls how auto-placed items fill the grid. Use dense to fill gaps.

Pro Tips

  • Use gap instead of margins for spacing - it's cleaner and handles edge cases
  • Combine Grid for overall page layout with Flexbox for component interiors
  • Use minmax() for responsive grids that don't break on small screens
  • auto-fit and auto-fill are powerful for responsive grids without media queries
  • Set flex: 1 as shorthand for flex-grow: 1; flex-shrink: 1; flex-basis: 0%
