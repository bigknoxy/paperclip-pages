## Color Theory Basics for Web Designers

Understanding a few color theory concepts will help you use these tools more effectively:

### The Color Wheel

The color wheel organizes colors by their relationship to each other:
- **Primary:** Red, blue, yellow
- **Secondary:** Orange, green, purple
- **Tertiary:** Mixtures of primary and secondary

### Color Harmonies

- **Complementary:** Colors opposite each other on the wheel (high contrast)
- **Analogous:** Colors next to each other (harmonious, low contrast)
- **Triadic:** Three evenly spaced colors (balanced, vibrant)
- **Monochromatic:** Variations of a single color (sophisticated, cohesive)

### The 60-30-10 Rule

For balanced web design:
- **60%** dominant color (backgrounds, main areas)
- **30%** secondary color (sidebars, cards)
- **10%** accent color (buttons, calls-to-action)

## Best Practices for Using Color Palette Generators

### 1. Start with Your Brand

If you have existing brand colors, lock those first in tools like Coolors or Huemint, then generate complementary colors.

### 2. Test for Accessibility

Always check contrast ratios. Most palette generators include accessibility checkers—use them. Your text should have at least a 4.5:1 contrast ratio against its background.

### 3. Consider Context

A palette that looks great on a portfolio site might not work for an e-commerce checkout page. Always test in context.

### 4. Create Variations

Generate multiple palettes for different states:
- Default state
- Hover states
- Active/selected states
- Error and success states

### 5. Document Your Choices

Save your final palette with notes about where each color should be used. This maintains consistency as your project grows.
