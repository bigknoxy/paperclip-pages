Thank you for contributing to Paperclip — we appreciate your time and energy.

How to get started
- Read `tasks/todo.md` to find small, high-value tasks.
- Run the project locally:
  1. `npm install`
  2. `npm run dev` — starts the Vite dev server
  3. `npm test` — runs unit tests (Vitest)

Branching & PRs
- Create a descriptive branch: `feature/<short-description>` or `fix/<short-description>`.
- Open a pull request against `main` with a short summary and testing notes.
- Keep PRs small and focused; one change per PR when possible.

Commit messages
- Use present-tense, short subject lines (e.g. `feat: add greet util`).

Code style
- Follow existing patterns in the repo. Use simple, readable code. Add tests for logic changes.

Testing
- Add the smallest test that reproduces the bug or validates the feature.

Questions or help
- If you need help, open an issue and tag `help wanted` in the title.
